---
title: "The Vulnerability Manifesto"
---

There is a test that settles most arguments in supply-chain security, and almost no one applies it. Ask of any control you have been sold: *would it still be true if the thing had never been invented?*

Run that test on the waiting periods, on the catalog numbers, on the generic scanners, and the same answer falls out every time. The protection was never the timer, the number, or the brand on the invoice. It was **detection** — and we keep bolting it on everywhere except the one place it would actually stop an attacker.

This is a short book about putting it back where it belongs: at the source, at publish, in front of the attacker rather than behind him.
